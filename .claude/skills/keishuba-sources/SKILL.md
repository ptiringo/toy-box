# keishuba-sources（軽種馬登録の典拠マップと一次資料の読み方）

`horseracing` コンテキストで JAIRS 管掌の登録（血統登録・馬名登録・繁殖登録・繁殖成績報告）を
モデリングするときの **一次資料の所在と読み方** を集約する。制度の事実を典拠付きで書く前に、ここから
原典に当たる。

このスキルの役割は **「どこに・何が・どう読めば書いてあるか」までで、確定した不変条件そのものは持たない**。
品種5区分・分娩語彙・依存チェーンなどの検証済み知見は、原典更新やモデル変更で乖離しないよう
Issue / ADR / 用語集を唯一の出所とし、ここからはリンクで辿る（後述「検証済み知見の所在」）。

> **大原則（[[feedback_primary_source_citations]]）**: 仕様・不変条件・分類を典拠付きで残す前に、
> WebSearch や小モデル要約を鵜呑みにせず原典（規程・条文・様式 PDF）に当たり、**条番号・ページまで
> 確認してから書く**。WebSearch は探索の方向当たりに留める。実際、品種区分を検索要約の「甲乙丙」で
> issue に書いて原典精読で誤りが判明し訂正した実績がある（旧区分。現行は別表の5区分）。

## 1. 権威ソースの区別（混同しやすい・重要）

| 機関 | 正式 | 役割 | このドメインでの扱い |
| --- | --- | --- | --- |
| **JAIRS** | 公益財団法人 ジャパン・スタッドブック・インターナショナル | **登録機関**（血統・繁殖・馬名登録、証明書発行、Stud Book 編纂） | `horseracing` の中核。本スキルの主対象 |
| **JBBA** | 公益社団法人 日本軽種馬協会（jbba.jp） | 生産振興・種牡馬繋養（種馬場運営・種付サービス・研修・せり） | **登録機関ではない運用側**。JBBA の「種牡馬」は商用繋養の意味で、JAIRS の「種牡馬の繁殖登録」とは別レイヤー |
| **JRA / NAR** | 日本中央競馬会 / 地方競馬全国協会 | **競走馬登録**（中央／地方） | JAIRS とは別の境界づけられたコンテキスト（ADR-0013）。当面スコープ外。下記「JRA 境界の最小参照」 |

- 「種牡馬の繁殖登録」は **JAIRS**。JBBA の繋養とは別。
- 繁殖登録は同一個体への血統登録を前提としたロール付与であり、JAIRS が登録機関。

## 3. 一次資料の読み方（取得・レンダリング手順）

和文 JAIRS PDF は **CID フォントでテキスト層が無い**ことが多く、`pdftotext` でも WebFetch の内部モデルでも
読めない（「破損/暗号化」と誤報告される）。**文字は画像として読む**のが確実（[[feedback_reading_pdfs]]）。

- **小さめ PDF（≤10ページ）**: WebFetch が保存したパス、または URL を直接 `Read`（harness が画像描画する）。
- **大容量／多ページ／CID（実施基準など）**:
  ```bash
  curl -sL -o f.pdf https://www.jairs.jp/contents/pdf/tourokukiteijisshikijyun.pdf
  pdftoppm -png -r 150 -f <開始p> -l <終了p> f.pdf out/p   # 自前で PNG 化
  ```
  生成した PNG を `Read`。小字が判読しづらいときは `-r 300` で再レンダリング。
- **全文精読**: ページ範囲を分割して `general-purpose` サブエージェントを並列起動し、各自 pdftoppm＋Read で
  読ませ、**典拠を「条番号＋ページ」で返させる**。実施基準49pを6エージェントで網羅した実績あり。
- **poppler（pdftoppm）が無い場合**: brew は sandbox＋ディレクトリ権限で session 内インストール不可。
  別ターミナルで導入してもらう。`pdftotext` は CID で無力なので不要。

> JRA 系 PDF は様式が異なり **`pdftotext` でテキスト抽出可（画像化不要）**。和文でも JAIRS とは扱いが違う点に注意。

## 4. JRA 境界の最小参照（スコープ外・境界確認用）

競走馬登録（JRA）は JAIRS 中心の `horseracing` とは **別の境界づけられたコンテキスト**（ADR-0013、当面スコープ外）。
依存の向き（JAIRS が上流）を確認するときだけ参照する。深入りは ADR-0013／#351 のリサーチへ。

- 典拠の所在: `https://www.jra.go.jp/company/about/law/pdf/` 配下（**pdftotext でテキスト抽出可**）。
  - `01.pdf` = 競馬法（第14条＝登録馬でなければ中央競馬に出走不可）
  - `07.pdf` = 日本中央競馬会競馬施行規程（第3章 競走馬登録＝第17〜29条が本体）
- 依存チェーン: **血統登録(JAIRS) → 馬名登録(JAIRS) → 馬主登録(JRA) ・預託契約 → 競走馬登録(JRA)**。
  競走馬登録申請には JAIRS 発行の血統証明書添付が必須で、馬名は血統登録原簿の馬名と一致が要件。
- 中央(JRA)と地方(NAR)の登録は排他。繁殖供用馬は競走馬登録不可（繁殖ロールと競走ロールは同時併存不可）。

## 5. 検証済み知見の所在（リンクで辿る・転記しない）

原典精読で確定した不変条件・語彙・分類は、ここに転記すると乖離するため **下記が唯一の出所**。
書く前にこれらと突き合わせ、足りなければ原典に当たって更新する。

- **用語の定義・別名・禁止語**: `docs/ubiquitous-language.md`（horseracing コンテキスト節）。
  ビルディングブロックは「コードが唯一の出所」で `UbiquitousLanguageCatalogTest` が同期を担保。
- **設計判断（コードから読めない経緯）**: `docs/adr/`。とくに
  - ADR-0013 競走馬登録を別コンテキストに（JRA 境界）
  - ADR-0014 集約をまたぐ前提条件は自己検証ファクトリで検証（生成口封じ込めは撤廃）
  - ADR-0016 / ADR-0018 / ADR-0020 分娩結果「産駒なし」「種付せず」のモデリング
  - ADR-0021 父母不在は 422 / ADR-0022 集合制約の検証にドメインサービスがリポジトリポートを読み取り利用
- **原典精読で確定した事実・不変条件**: GitHub Issue のコメント。検索は `gh issue view <n>` や
  `gh search issues`。主な収集先:
  - **#265** 繁殖成績報告（分娩結果の語彙＝様式14号裏／報告様式13号・14号／翌年5/31返送）
  - **#266** 繁殖登録（雄雌共通の単一登録・証明書の性で区別）
  - **#267** 父母不明・輸入馬の血統登録経路（揚陸日起算）
  - **#277** 品種区分（**現行は別表p.10-13 の5区分＝サラブレッド/アラブ/アングロアラブ/サラブレッド系種/アラブ系種＋アラブ血量%閾値**。検索要約の「甲乙丙」は旧区分で誤り）
  - **#312–#320** 原典精読から起票した派生サブドメイン（個体識別・証明書ライフサイクル・異動/所有権・課金・種付有効性・毛色親子整合・繁殖供用停止・国際移動・Stud Book 編纂）。すべて条番号・ページ付き
- **ドメインモデリングの現状と次の宿題**: メモリ [[project_keishuba_modeling]]（point-in-time。コードと突き合わせて使う）。

## やってはいけないこと

- 検索要約・記憶だけで品種区分や分娩語彙など制度の事実を確定的に書く（必ず原典＋条番号/ページで裏取り）。
- JAIRS / JBBA / JRA を混同する（登録機関は JAIRS、JBBA は繋養運用、競走馬登録は JRA で別コンテキスト）。
- 確定した不変条件をこのスキルに転記して二重管理する（用語集・ADR・Issue へリンクで逃がす）。
- 大容量・CID PDF を WebFetch / pdftotext で読もうとして「破損」と諦める（curl＋pdftoppm＋画像 Read）。
