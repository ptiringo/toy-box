# sakamichi-sources（坂道シリーズの権威ソースマップと用語の区別）

`sakamichi`（エンターテイメント）コンテキストで坂道シリーズ（乃木坂46・櫻坂46・日向坂46）を
モデリングするときの **権威ソースの所在と用語の区別** を集約する。ドメイン事実を典拠付きで書く前に、
ここから公式の出所に当たる。

軽種馬登録の [keishuba-sources](../keishuba-sources/SKILL.md) と役割は同じだが、坂道には規程・条文のような
一次資料が薄い。**権威ソース＝各グループの公式サイト（プロフィール・お知らせ）であり、加入・卒業・改名の
公式発表（お知らせ記事）が事実の起点**になる。

このスキルの役割は **「どこに・何が・どう読めば書いてあるか」までで、確定した可変事実そのものは持たない**。
在籍者・期生数・選抜編成は時点依存で陳腐化が速いため、ここに転記せず公式サイト／用語集／Issue を出所とし
リンクで辿る（後述「検証済み知見の所在」）。

> **大原則**: 制度の事実（系譜・用語・構成）を確定的に書く前に、WebSearch の要約や自分の記憶を鵜呑みにせず
> **各グループ公式サイトのお知らせ／プロフィールに当たってから書く**。LLM の記憶は改名日・期生数・在籍で
> 誤りやすく、URL も `www` 有無・末尾を取り違えやすい（規則性からの推測で確定 URL を書かない）。

## 1. 権威ソースの階層（混同しやすい・重要）

| 階層 | 出所 | 扱い |
| --- | --- | --- |
| **一次（典拠）** | 各グループ**公式サイト**（メンバー一覧・プロフィール・ニュース/お知らせ）、公式 YouTube/SNS の運営アナウンス、運営会社・ソニーミュージック系レーベルの発表 | **事実の起点**。加入・卒業・改名は公式お知らせを典拠にする |
| **二次（索引のみ）** | 日本語 Wikipedia、エケペディア（48pedia） | 全体像・索引には有用だが**典拠にしない**。必ず一次へ当たる |
| **参照しない** | ファンサイト・まとめ（anosaka / zaka46db 等）・個人ブログ・SNS の噂/リーク | 方向当たりにも慎重に。確定記述の典拠にしない |

## 2. グループと系譜（典拠マップ）

坂道シリーズの版元・運営元はソニー・ミュージック（秋元康プロデュース）。**本コンテキストの主対象は次の
アイドル3グループ**。

| グループ | 公式サイト | 運営会社 |
| --- | --- | --- |
| 乃木坂46 | `https://www.nogizaka46.com/` | 乃木坂46合同会社 |
| 櫻坂46 | `https://sakurazaka46.com/` | Seed & Flower 合同会社 |
| 日向坂46 | `https://www.hinatazaka46.com/` | Seed & Flower 合同会社 |

※ URL・運営は WebSearch で確認した 2026-06-28 時点。リンク切れ・改訂時は各グループ名で公式を引き直す。
乃木坂46 と櫻坂/日向坂で**運営会社が異なる**点に注意（同一運営と決め打たない）。

### 系譜（改名マップ）

- **欅坂46 →（2020-10-14）櫻坂46**: 欅坂46 は 2020-10-13 の THE LAST LIVE で活動終了、翌日から櫻坂46。
  字体は旧字「**櫻**坂46」が公式（「桜坂」は誤記）。
- **けやき坂46 →（2019-02-11）日向坂46**: けやき坂46（通称「ひらがなけやき」）は欅坂46 のアンダー/姉妹として
  発足し、日向坂46 に改名・独立。
- 2018 年の**坂道合同オーディション**合格者に**坂道研修生**（未配属プール）があり、後に各グループへ配属。

> **改名をまたぐ同一性は設計判断であって既定事実ではない**。「グループ名が変わっても人・期生の同一性が
> 継続するか／別グループ実体として扱うか」「期生番号が改名後も連番か」は**モデリングで決めること**。
> 記憶で「継続する」と断定せず、必要なら公式お知らせで裏取りし、判断は ADR / Issue に残す。

### 外延（境界づけ）

狭義＝上記3グループ。次は**性質が異なるため安易にスコープへ混ぜない**（含めるかは最初に境界を決める）:

- **吉本坂46**: 吉本興業との合同・男女混成で運営レイヤーが別。
- **坂道研修生**: 未配属の過渡的プール。
- **AKB48 グループとの兼任**: 「兼任」は主に AKB 系との概念（別フランチャイズ）。坂道内では稀で、
  これを「2グループ同時所属」として一般化すると過剰設計になりうる。

## 3. 一次資料の読み方（取得手順）

坂道の公式は HTML サイト（JAIRS の PDF と違いテキストで読める）。ただし **これらのホストは Bash
サンドボックスの許可ホストに無く、素の `curl` / `wget` は届かない**。取得は Claude ツールで行う:

- **WebSearch**: グループ名・「お知らせ」「メンバー」等で方向当たり（Anthropic 経由で到達可）。要約は鵜呑みにしない。
- **WebFetch**: 公式ページの URL を直接渡して本文を読む（同上、サンドボックス外）。
- **イベント（加入・卒業・改名）の典拠**: 各公式サイトの **ニュース/お知らせ記事**。日付つきの運営発表を引く。
- **現在の在籍者・期生**: 公式サイトの **メンバー一覧/プロフィール**。時点依存なので参照日を添える。

> 許可ホストを足したい場合は各自のローカル設定（`.claude/settings.local.json`）で行い、共有ファイルには書かない
> （CLAUDE.md「Claude 指示ファイル・スキルの記述方針」）。

## 4. 用語の区別（混同しやすい）

| 用語 | 意味 | 混同しやすい点 |
| --- | --- | --- |
| グループ | 乃木坂46 / 櫻坂46 / 日向坂46 | 改名前後（欅坂↔櫻坂、けやき坂↔日向坂）を同一視/二重計上しない |
| メンバー | 集約ルート候補（`Member`） | — |
| 期生（きせい） | 加入時期で区切るコホート（1期生/2期生…）。加入時に固定 | **番号はグループごとに独立採番**（横断で一意でない） |
| 加入 / 卒業 | メンバーの参加 / 円満離脱 | 契約解除・活動辞退は「卒業」と別概念になりうる（設計判断） |
| 選抜（せんばつ） | **シングル表題曲**を歌う選ばれたメンバー集合。⇔ アンダー（非選抜） | グループの恒久属性ではなく**シングル単位の一時的編成**。メンバーと混同しない |
| センター / フォーメーション | 選抜の中心人物 / 立ち位置 | シングルごとに変わる |
| 改名 | グループ名変更（欅坂46→櫻坂46 等） | **個人（メンバー）の改名**とは別概念 |
| 兼任 | 複数グループ同時所属 | 主に AKB 系との概念で坂道内は稀（§2 外延参照） |

- 漢字「**欅坂**46」と ひらがな「**けやき坂**46」は**別グループ**（典型的な取り違え）。
- 「46」はメンバー数ではない（命名上の含意）。

## やってはいけないこと

- 記憶や WebSearch 要約だけで改名日・期生数・在籍者・選抜を確定的に書く（公式お知らせ/プロフィールで裏取り）。
- 公式 URL を規則性から推測して書く（`www` 有無・末尾を取り違える。確認した URL のみ載せる）。
- 欅坂46 と けやき坂46 を混同する／櫻坂を「桜坂」と書く。
- 改名をまたぐ同一性・期生連番を既定事実として断定する（モデルの設計判断。ADR/Issue に残す）。
- 吉本坂46・坂道研修生・AKB 兼任を境界を決めずにスコープへ混ぜる。
- 確定した可変事実をこのスキルに転記して二重管理する（公式・用語集・Issue へリンクで逃がす）。
- これらのホストを素の `curl` で取ろうとして「届かない」と諦める（WebSearch / WebFetch を使う）。
