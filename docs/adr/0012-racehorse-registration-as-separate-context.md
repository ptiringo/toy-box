# 0012. 競走馬登録(JRA)を JAIRS 中心ドメインから境界づけられたコンテキストとして分離する

- Status: Accepted
- Date: 2026-06-21
- Deciders: Matsui

## Context（背景・課題）

本プロジェクトの軽種馬登録ドメインは、これまで **JAIRS（ジャパン・スタッドブック・インターナショナル）管掌の登録**（血統登録・繁殖登録・馬名登録）を `horseracing` コンテキストで探索的にモデリングしてきた。

ここに #266 で **競走馬登録（Racehorse）** を追加し、`Stallion / Broodmare / Racehorse` を「`BloodHorse`（血統登録された個体）が担う対称なロール」として統一しようとした（PR #348、当時の ADR-0012 案＝ロールを別集約とする方針）。しかしこの実装は **競走馬登録(JRA 管轄)の前提条件を一次資料で確認せず推測（「馬名登録済みが前提」程度）で決めていた**ため、典拠不足を理由に PR・ADR 案ごとクローズした。

その前提を埋めるため #351 で JRA の一次資料を精読した（[JRA 法令一覧](https://www.jra.go.jp/company/about/law/) 配下の PDF。`curl → pdftotext` でテキスト抽出可能）。判明した事実は以下。

- **法的根拠は競馬法第14条**: 「日本中央競馬会が行う登録を受けた馬でなければ、中央競馬の競走に出走させることができない」。具体手続は **日本中央競馬会競馬施行規程 第3章 競走馬登録（第17〜29条）** が定める。
- **依存チェーン**: 血統登録(JAIRS) → 馬名登録(JAIRS) → 馬主登録(JRA, 競馬法第13条)・調教師との預託契約 → **競走馬登録(JRA)**。競走馬登録の申請には JAIRS 発行の血統証明書の添付が必須（施行規程第18条2項）であり、馬名は血統登録原簿の馬名と一致していなければならない（同第22条(7)）。
- **管掌組織が別**: 競走馬登録は **JRA（中央競馬）** の登録で、登録簿（競走馬登録簿）も審査も JRA 側にある。JAIRS の血統登録・繁殖登録・馬名登録とは別組織・別原簿。地方競馬は「地方馬登録」（NAR）という排他的な別登録（施行規程第21条(10)・第24条(2)）。
- **ロールの非対称性**: 繁殖の用に供された馬は競走馬登録を受けられず（施行規程第21条(3)）、用途転用は抹消事由（同第24条(8)）。すなわち繁殖ロールと競走ロールは**同時併存できない**。さらに繁殖ロール（Stallion/Broodmare）の登録根拠は **繁殖登録（JAIRS、現行ドメインと同一コンテキスト）** であるのに対し、競走ロール（Racehorse）の登録根拠は **競走馬登録（JRA、別コンテキスト）** にある。

この非対称性こそ #266 が躓いた本質である。3 つを「同じ `BloodHorse` の同列ロール」として対称に扱うと、`horseracing`（JAIRS）コンテキストの中に JRA 管掌の概念が混入し、境界が曖昧になる。

### 検討した代替案

- **(却下) 3 ロールを `horseracing` 内で対称に統一する**（#266 当初案）。管掌組織と原簿が異なる競走馬登録を JAIRS コンテキストに混ぜることになり、境界づけられたコンテキストの分離（ArchUnit で強制）と整合しない。一次資料に照らして概念的にも不正確。

## Decision（決定）

**競走馬登録(JRA)は、JAIRS 中心の `horseracing` コンテキストとは別の境界づけられたコンテキストに属する概念として扱う。** 現行ドメインの `BloodHorse` の同列ロールとして `Racehorse` を統合しない。

- `horseracing`（JAIRS）コンテキストが担うのは血統登録・馬名登録・繁殖登録、および繁殖ロール（Stallion/Broodmare）まで。
- 競走馬登録を実装する場合は、中央競馬（`racing` 等）を独立した境界づけられたコンテキストとして新設し、血統登録個体への参照は `BloodHorseId`（コンテキスト跨ぎの ID 参照）で持つ。集約内ロールとしては表現しない。
- **本 ADR はスコープ境界の確定までを決定対象とする**。競走馬登録を実際に「いつ・どう実装するか」（当面スコープ外とするか、`racing` コンテキストを新設して着手するか）は本 ADR では確定せず、別途の判断・別 ADR に委ねる。

## Consequences（結果・影響）

- **良くなる点**: コンテキスト境界が一次資料の管掌区分（JAIRS / JRA / NAR）と一致し、`horseracing` コンテキストが JAIRS 登録に閉じる。繁殖ロール（Stallion/Broodmare, #318 等）は競走馬登録の確定を待たずに、既存 `BreedingRegistration` を先例として進められる。
- **引き受けるトレードオフ**: 競走馬登録を扱う段階で新コンテキストの配線（パッケージ・ArchUnit のコンテキスト分離対象への追加・Repository ポート等）が別途必要になる。`BloodHorse` と競走馬登録の関係は集約内参照ではなく ID 参照になるため、両者を跨ぐ整合（繁殖供用馬は競走馬登録不可、等）はドメインサービスまたはアプリケーション層で担保する。
- **関連**: 調査の典拠と詳細は #351（コメントに一次資料の条番号付きで整理）、経緯は #266 / クローズ済み PR #348。ロールを「BloodHorseId 参照の別集約＋生成口 internal ＋付与サービスへの封じ込め」で表す方針自体は [ADR-0010](0010-confine-aggregate-creation-to-domain-service.md) と既存 `BreedingRegistration` が先例として有効。
